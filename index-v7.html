📊 数据流向说明

🔵 KB Update（知识库更新）

从反馈闭环到知识库的更新流，包括标签优化、规则更新、新菜谱入库

🟢 Policy Update（策略更新）

从执行结果到约束引擎的策略调整，优化排序和推荐逻辑

🟡 KB Update（映射库更新）

需求↔变量↔旋钮映射关系的持续优化和校准

🎯 核心交付物

  • RDS (Recipe Demand Spec) - 需求端统一语言，将用户需求标准化为可处理的维度表达
  • RecipeProfile - 交付端统一语言，将菜谱解析为设备可执行的标准化结构
  • Mapping/Knobs Library - 需求↔交付映射规则库，12 个可调控制变量
  • Executable Recipe Package - 可执行菜谱包，包含标准 SOP 和设备指令

🔄 闭环优化流程

  • 执行数据回传 - 出品时间、退菜率、用户评分、空盘率
  • 异常检测 - 识别步骤异常、重复输入、质量风险
  • 标签更新 - 基于实际表现优化菜谱标签
  • 规则优化 - 调整映射规则和推荐策略
  • 个性化推荐 - 基于用户反馈的个性化菜谱推荐

  • ✅ V6 已完成 - 菜谱库 (13,510 条) + 维度网络 + 瀑布流详情
  • ✅ 数据治理 - RecipeProfile 解析器 + 去重 + 特征计算

🔧 技术栈说明

  • 输入层 - ASR(语音识别) + CV(图像分类) + LLM(文本理解)
  • 检索层 - 标签过滤 + 向量相似度 (ANN) + 规则匹配
  • 约束引擎 - 设备能力检查 + 时长约束 + 质量风险评估
  • 生成层 - 参数化菜谱生成 + 设备指令打包
  • 执行层 - Kitchen OS + 设备控制 + 数据回传
